## Introduction

Next.js has revolutionized the way we build React applications. With the release of Next.js 14 and the App Router, developers now have unprecedented control over performance, routing, and data fetching.

## Why Next.js?

Here are some compelling reasons to choose Next.js:

1. **Server-Side Rendering (SSR)**: Improved SEO and performance
2. **Static Site Generation (SSG)**: Lightning-fast page loads
3. **File-based Routing**: Intuitive and automatic routing
4. **API Routes**: Built-in API endpoints
5. **Image Optimization**: Automatic image optimization
6. **TypeScript Support**: First-class TypeScript integration

## Getting Started

Install Next.js with the following command:

```bash
npx create-next-app@latest my-app
cd my-app
npm run dev
```

## Project Structure

A typical Next.js 14 project structure looks like this:

```
my-app/
├── app/
│   ├── layout.tsx
│   ├── page.tsx
│   └── api/
├── components/
├── public/
└── package.json
```

## Building Your First Page

Create a new page in the `app` directory:

```tsx
// app/about/page.tsx
export default function AboutPage() {
  return (
    <div>
      <h1>About Us</h1>
      <p>Welcome to our Next.js application!</p>
    </div>
  );
}
```

## Server and Client Components

Next.js 14 introduces React Server Components by default:

```tsx
// Server Component (default)
async function ServerComponent() {
  const data = await fetchData();
  return <div>{data}</div>;
}

// Client Component (use 'use client' directive)
'use client';
import { useState } from 'react';

function ClientComponent() {
  const [count, setCount] = useState(0);
  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
```

## Data Fetching

Fetch data directly in Server Components:

```tsx
async function Posts() {
  const posts = await fetch('https://api.example.com/posts', {
    next: { revalidate: 3600 } // Revalidate every hour
  });
  const data = await posts.json();
  
  return (
    <ul>
      {data.map(post => (
        <li key={post.id}>{post.title}</li>
      ))}
    </ul>
  );
}
```
